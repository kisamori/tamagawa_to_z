# 考古学遺跡類似度分析システム 使用ガイド

## 概要

このシステムは、**地名の空間分布パターンを機械学習で分析し、考古学遺跡の類似度スコアを算出する**革新的なツールです。ブラジル・アクレ州の既知遺跡データを基に開発され、新しい候補地点の考古学ポテンシャルを定量的に評価できます。

## 🎯 分析の目的と意義

### なぜ地名分析なのか？

考古学において、**地名（トポニム）は過去の人間活動の痕跡**です：

1. **水系関連地名**：「Rio（川）」「Igarapé（小川）」→ 水源近くの居住地
2. **地形関連地名**：「Serra（山）」「Campo（平野）」→ 地形的特徴を活かした立地
3. **植生関連地名**：「Mata（森）」「Floresta（森林）」→ 自然資源へのアクセス
4. **交通関連地名**：「Estrada（道路）」「Caminho（小道）」→ 交通の要衝

**既知の考古学遺跡周辺にどのような地名が分布しているか**を分析することで、**古代人が好んだ立地パターン**を数値化できます。

### 従来手法の限界と本システムの革新性

**従来の考古学調査**：
- 😞 主観的判断：「川の近くは良さそう」「高台は住みやすそう」
- 😞 経験依存：ベテラン研究者の勘に頼る
- 😞 定性的評価：「高い」「低い」といった曖昧な表現
- 😞 地域特化：他地域への応用が困難

**本システムの革新**：
- ✅ **客観的分析**：機械学習による数値化
- ✅ **データ駆動**：既知遺跡1,438件の地名データを活用
- ✅ **定量的評価**：0-1の連続値でスコア化
- ✅ **汎用性**：他地域・他文化圏への適用可能

## 🔬 分析の仕組み詳細

### ステップ1: 地名データの収集と分類

```
遺跡周辺3km圏内の全地名を抽出
     ↓
7つのカテゴリーに自動分類
```

| カテゴリー | 例 | 考古学的意味 |
|------------|----|--------------| 
| **waterway** | Rio Amazonas, Córrego Pequeno | 水源アクセス、交通路 |
| **natural** | Serra do Norte, Mata Densa | 自然資源、地形的優位性 |
| **place** | Vila Central, Povoado Santo | 集落形成の核 |
| **highway** | Estrada BR-364, Rua Principal | 交通インフラ、移動路 |
| **landuse** | Fazenda Santa Clara, Área Rural | 土地利用、農業活動 |
| **man_made** | Torre de Água, Ponte Nova | 人工構造物、技術レベル |
| **other** | その他の地名 | - |

### ステップ2: 特徴量エンジニアリング（56の数値指標生成）

各遺跡について、地名分布から**56個の数値特徴量**を自動生成：

#### A. 空間分布特徴量
```
距離ベース特徴量:
- 最寄り地名までの距離（カテゴリー別）
- 平均距離、中央値距離、標準偏差

圏域別カウント:
- 1km圏内の地名数（カテゴリー別）
- 2km圏内の地名数（カテゴリー別）  
- 3km圏内の地名数（カテゴリー別）

密度特徴量:
- 各圏域での地名密度（件/km²）
```

**例**: Quixadá遺跡の場合
- waterway_count_1km: 2件（1km圏内に川関連地名が2つ）
- highway_density_2km: 15.2件/km²（2km圏内の道路密度）
- mean_distance_natural: 1.8km（自然関連地名までの平均距離）

#### B. 方位分布特徴量
```
方位統計:
- 地名の方位角平均・標準偏差
- 最大方位角範囲

象限分析:
- 北東象限（0-90°）の地名数
- 南東象限（90-180°）の地名数
- 南西象限（180-270°）の地名数  
- 北西象限（270-360°）の地名数

方位エントロピー:
- 地名分布の方位的均等性
```

**意味**: 古代人が**特定の方向を好んだか**（例：南向き斜面、風上側）を数値化

#### C. 地名カテゴリー特徴量
```
タイプ比率:
- waterway地名の割合（%）
- highway地名の割合（%）
- natural地名の割合（%）
- ...各カテゴリーの構成比

河川距離特徴量:
- 最寄り河川までの距離
- 河川方向の角度統計

統計指標:
- 距離分布の歪度（偏り）
- 距離分布の尖度（とがり具合）
```

### ステップ3: データ前処理と特徴量選択

```
56特徴量の正規化（StandardScaler）
     ↓
高相関特徴量の除去（r > 0.95）
     ↓
39特徴量に絞り込み
```

**なぜ正規化？**: 「地名数（0-300）」と「割合（0-1）」では単位が違うため、機械学習で適切に比較するため

**なぜ特徴量選択？**: 「地名総数」と「3km圏内地名数」は完全に相関（r=1.000）するため、冗長な情報を除去

### ステップ4: 機械学習による類似度計算

4つの異なるアルゴリズムで多角的に評価：

#### 1. k最近傍法（k-NN）
```python
# 39次元特徴量空間での距離計算
knn = NearestNeighbors(n_neighbors=3)
distances, indices = knn.fit(features).kneighbors(target_site)
similarity_score = 1 / (1 + mean(distances))
```
**意味**: 「**この遺跡と最も似ている既知遺跡との距離**」を計算。距離が近い = 類似度高い

#### 2. クラスタリング（K-means）
```python
# 既知遺跡を3つのグループに分類
kmeans = KMeans(n_clusters=3)
cluster_centers = kmeans.fit(features).cluster_centers_
distance_to_center = min(distances_to_all_centers)
similarity_score = 1 / (1 + distance_to_center)
```
**意味**: 「**どの遺跡グループの中心に最も近いか**」を評価。グループの典型例に近い = 類似度高い

#### 3. ガウシアン混合モデル
```python
# 確率分布による適合度計算
gmm = GaussianMixture(n_components=3)
log_probability = gmm.fit(features).score_samples(target_site)
similarity_score = exp(log_probability)
```
**意味**: 「**既知遺跡の分布パターンにどれだけ当てはまるか**」を確率で評価

#### 4. 異常検知（Isolation Forest）
```python
# 正常パターンからの逸脱度計算  
iso_forest = IsolationForest()
anomaly_score = iso_forest.fit(features).decision_function(target_site)
similarity_score = (anomaly_score + 1) / 2  # 0-1に正規化
```
**意味**: 「**既知遺跡パターンから外れていないか**」を評価。正常 = 類似度高い

### ステップ5: 複合スコア計算

```python
複合スコア = k-NN × 0.35 + クラスタ × 0.25 + ガウシアン × 0.25 + 異常度 × 0.15
```

**重み付けの根拠**:
- **k-NN（35%）**: 最も直感的で信頼性の高い類似度指標
- **クラスタ（25%）**: グループ全体の典型性を評価
- **ガウシアン（25%）**: 確率的モデルによる客観性
- **異常度（15%）**: 外れ値検出による安全性確保

## 🔍 機械学習アルゴリズムの役割分担

### なぜ4つのアルゴリズムを組み合わせるのか？

単一のアルゴリズムでは**見落としやバイアス**が発生する可能性があります：

#### ❌ k-NN単体の限界
```
問題: 外れ値に敏感
例: 1つの遺跡が特殊な立地だと、その近傍も「類似」と判定される
→ 誤った高スコアを付与する可能性
```

#### ❌ クラスタリング単体の限界  
```
問題: グループ数（k=3）の事前設定が必要
例: 実際は2グループや4グループが適切かもしれない
→ 不自然な分類による評価ミス
```

#### ✅ 4つの組み合わせによる相互補完
```
k-NN: 「最も近い遺跡」との比較
↓
クラスタ: 「グループの典型例」との比較  
↓
ガウシアン: 「確率分布」による客観評価
↓
異常検知: 「外れ値」でないことの確認
↓
重み付き平均: バランス良い最終判定
```

### 各アルゴリズムが検出する遺跡タイプ

| アルゴリズム | 高スコアを付与する遺跡タイプ | 検出例 |
|--------------|---------------------------|---------|
| **k-NN** | 既知遺跡と酷似した立地 | 「Quixadá遺跡とほぼ同じ地名パターン」 |
| **クラスタ** | グループの代表的特徴を持つ | 「川沿い農業地域群の典型例」 |
| **ガウシアン** | 統計的に標準的な立地 | 「アクレ州遺跡の一般的パターン」 |
| **異常検知** | 外れ値でない正常な立地 | 「特殊すぎない、堅実な立地選択」 |

## 💡 理論的背景と学術的意義

### 地理情報学×考古学の融合

本システムは以下の学際的理論に基づいています：

#### 1. **トポニミー（地名学）理論**
```
Zelinsky (1955): 「地名は文化的景観の化石である」
↓
古代の地名が現代まで継承される傾向
↓
遺跡周辺の地名パターンから古代人の認知マップを復元
```

#### 2. **立地選択理論（Location Theory）**
```
von Thünen (1826): 距離と土地利用の関係
↓
Weber (1909): 最小費用による立地選択
↓
考古学への応用: 古代人も効率的立地を選択していた
```

#### 3. **空間自己相関理論**
```
Tobler's First Law: 「近くのものは遠くのものより類似している」
↓
類似の環境条件 → 類似の人間行動 → 類似の遺跡立地
↓
地名分布の類似性 = 立地選択パターンの類似性
```

## 使用方法

### 1. 統合分析（地名抽出 + 類似度分析）

一から地名抽出を行い、類似度分析まで一貫して実行：

```bash
# 基本実行
poetry run python scripts/run_analyze_site.py --region acre --similarity-analysis

# 可視化も含めて実行
poetry run python scripts/run_analyze_site.py --region acre --similarity-analysis --visualize

# デバッグモードで実行
poetry run python scripts/run_analyze_site.py --region acre --similarity-analysis --debug
```

### 2. 類似度分析のみ（既存CSVファイル使用）

既存の地名分析CSVファイルを使用して類似度分析のみ実行：

```bash
# 基本実行
poetry run python scripts/run_similarity_analysis.py \\
    --csv-path data/output/site_analysis/site_analysis_YYYYMMDD_HHMMSS/site_toponym_analysis_acre.csv \\
    --region acre

# 出力ディレクトリを指定
poetry run python scripts/run_similarity_analysis.py \\
    --csv-path path/to/your/data.csv \\
    --output-dir custom/output/dir \\
    --region acre
```

### 3. Pythonコードでの使用

```python
from tamagawa_to_z.site_analysis import ArchaeologicalSimilarityAnalyzer

# 分析器初期化
analyzer = ArchaeologicalSimilarityAnalyzer('path/to/site_toponym_analysis.csv')

# 分析実行
analyzer.load_data()
analyzer.engineer_features()
analyzer.preprocess_features()
analyzer.remove_high_correlation_features()
analyzer.build_similarity_models()

# 類似度スコア計算
scores_df = analyzer.calculate_similarity_scores()
print(scores_df.sort_values('composite_score', ascending=False))
```

| gaussian_score | ガウシアン混合モデルによる類似度 |
| anomaly_score | 異常検知による正常度 |

## 技術仕様

### 機械学習アルゴリズム

1. **k最近傍法 (k-NN)**: 特徴量空間での近傍距離による類似度
2. **クラスタリング**: K-meansクラスタ中心からの距離
3. **ガウシアン混合**: 確率分布による適合度
4. **異常検知**: Isolation Forestによる正常度評価

### 複合スコア計算式

```
複合スコア = kNN×0.35 + クラスタ×0.25 + ガウシアン×0.25 + 異常度×0.15
```

### 特徴量

- **総計56特徴量** → **相関除去後39特徴量**
- 空間分布特徴量：距離統計、圏域別カウント、密度
- 方位分布特徴量：角度統計、象限分析、エントロピー
- 地名カテゴリー特徴量：タイプ比率、河川距離

## トラブルシューティング

### よくある問題

1. **ImportError: No module named 'seaborn'**
   ```bash
   poetry install  # 依存関係を再インストール
   ```

2. **FileNotFoundError: CSVファイルが見つかりません**
   - 正しいパスを指定しているか確認
   - `run_analyze_site.py`で事前に地名分析を実行

   - より少ない遺跡数で分析を実行
   - `--max-sites`オプションを使用
