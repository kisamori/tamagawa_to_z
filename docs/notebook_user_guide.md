# 非エンジニア向け Jupyter Notebook 操作ガイド

## 📖 概要

`notebooks/02_review_and_prompt.ipynb` は、アマゾン地名解析・遺跡候補抽出システムを非エンジニアでも簡単に操作できるように設計されたインターフェースです。

## 🚀 はじめ方

### 1. 環境準備

Jupyter Notebookを開く前に、以下を確認してください：

- **必要なデータファイル** が配置されていること
  - `data/raw/osm/norte-latest.osm.pbf` (OpenStreetMapデータ)
  - `data/raw/hydrorivers_sahydrorivers_sa/HydroRIVERS_v10_sa.shp` (河川データ)
  - `data/raw/GSW_occurrence/occurrence_70W_10Sv1_4_2021.tif` (水域データ)

- **OpenAI APIキー** が設定されていること
  - `.env` ファイルに `OPENAI_API_KEY=your_key_here` を記載

### 2. Notebook起動

```bash
# プロジェクトディレクトリに移動
cd /path/to/tamagawa_to_z

# Jupyter Notebook起動
jupyter notebook notebooks/02_review_and_prompt.ipynb
```

## 📋 操作手順

### ステップ1: 環境セットアップ
- 最初のセルを実行して、必要なライブラリとディレクトリを確認
- ✅ が表示されれば正常に準備完了

### ステップ2: パラメータ設定
- **解析地域 (BBox)**: 座標を指定（デフォルト: アクレ州西部）
- **河川距離閾値**: 現在の河川からどの程度離れた地点を候補とするか
- **水域頻度閾値**: 現在の水域頻度がどの程度低い地点を候補とするか
- **LLMサンプルサイズ**: コスト制御用（少なくするとコスト削減）

### ステップ3: ハーモナイザー実行
1. **🚀 ハーモナイザー実行** ボタンをクリック
2. 処理完了まで5-15分程度待機（実行時間が表示されます）
3. 必要に応じて **⛔ キャンセル** ボタンで処理を中断可能
4. ✅ 実行完了の表示と詳細統計を確認

### ステップ4: 結果確認
1. **🗺️ 地図表示** ボタンをクリック
2. 候補地点が地図上にマーカーで表示される
3. マーカーをクリックして詳細情報を確認

### ステップ5: AI診断
1. **🔍 AI診断実行** ボタンをクリック
2. AIが候補地点の品質を評価し、改善提案を生成
3. 診断完了まで2-5分程度待機（実行時間が表示されます）
4. 必要に応じて **⛔ キャンセル** ボタンで処理を中断可能
5. メトリクス（Recall@100, mAP等）の結果を即座に確認

### ステップ6: レポート確認
1. **📄 レポート表示** ボタンをクリック
2. 詳細な分析結果をMarkdown形式で確認
3. メトリクス（Recall@100, mAP等）をチェック

### ステップ7: 改善計画作成
1. **改善アクション** をドロップダウンから選択
2. **パラメータ** を編集（YAML形式）
3. **変更理由** を記入
4. **💾 計画保存** ボタンで保存

## 🎯 プリセット設定

効果的な設定のために、3つのプリセットを用意しています：

### 🎯 高精度設定
- **用途**: 確実性の高い候補のみを抽出
- **特徴**: 偽陽性（間違った候補）を減らす
- **適用場面**: 限られた時間で確実な地点を調査したい場合

### 🌐 高再現率設定  
- **用途**: できるだけ多くの候補を発見
- **特徴**: 見逃し（真の候補を逃す）を減らす
- **適用場面**: 網羅的な調査を行いたい場合

### ⚖️ バランス設定
- **用途**: 精度と再現率のバランス
- **特徴**: 標準的な設定
- **適用場面**: 最初の試行や一般的な調査

## 📊 メトリクスの見方

### Recall@100
- **意味**: 上位100候補に含まれる既知遺跡の割合
- **良い値**: 0.5以上（50%以上）
- **改善方法**: 閾値を緩くして候補数を増やす

### mAP (Mean Average Precision)
- **意味**: 候補の順位付けの精度
- **良い値**: 0.4以上
- **改善方法**: スコアリング手法の調整

### Workload（候補数）
- **意味**: 現地調査が必要な候補地点数
- **適正値**: 500件以下
- **調整方法**: 閾値を厳しくして候補数を減らす

## ❓ よくある質問

### Q: エラーが発生した場合は？
A: エラーメッセージを確認し、以下をチェック：
- データファイルが正しく配置されているか
- OpenAI APIキーが設定されているか
- インターネット接続が安定しているか

### Q: 処理に時間がかかりすぎる場合は？
A: 以下を試してください：
- **⛔ キャンセル** ボタンで処理を中断
- LLMサンプルサイズを小さくする
- ⚡ 水域頻度計算をスキップにチェック
- 解析地域を狭くする
- 実行時間が表示されるので、目安として参考にしてください

### Q: 候補地点が少なすぎる/多すぎる場合は？
A: パラメータを調整：
- **少なすぎる**: 閾値を緩くする（河川距離↑、水域頻度↓）
- **多すぎる**: 閾値を厳しくする（河川距離↓、水域頻度↑）

### Q: 保存されたYAMLファイルはどうやって使う？
A: 開発チームがパラメータを更新して次回実行に反映します。手動で編集することも可能です。
